# Implementation Plan: Corregir Eliminación/Edición + Filtros Avanzados de Participantes

**Branch**: `003-programs-fix-filters` | **Date**: 2026-04-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/003-programs-fix-filters/spec.md`

## Summary

Fix broken delete/edit operations on the Programs detail page (silent error handling, no `res.success` checks, native `confirm()` unreliable) and upgrade the ContactSelector component with advanced filtering capabilities (tag include/exclude, AND/OR mode, formula editor, date presets, device filter) matching the Contacts page, powered by switching to the `/api/contacts` endpoint which already supports all required filters.

**Testing**: Manual browser verification + Docker build

**Performance Goals**: Filter dropdown responsive (<200ms), search debounced at 500ms
**Constraints**: Docker-only builds, emerald/slate palette, minimal changes
**Scale/Scope**: 2 frontend files modified, 0 backend files

## Implementation Sequence

### Phase A: Fix Delete/Edit + Feedback (US1, US2, US4) — `programs/[id]/page.tsx`

1. **Add toast state** — `toastMessage`, `toastType`, auto-dismiss timer, render toast component
2. **Add confirmation dialog state** — `confirmAction` with message + onConfirm callback, render inline dialog
3. **Fix `handleDeleteProgram`** — Replace `confirm()` with confirmation dialog. Check `res.success` before navigating. Show toast on success/error.
4. **Fix `handleArchiveProgram`** — Check `res.success` before refreshing. Show toast.
5. **Fix `handleRemoveParticipant`** — Replace `confirm()` with confirmation dialog. Show toast.
6. **Fix `handleDeleteSession`** — Replace `confirm()` with confirmation dialog. Show toast.
7. **Add toast to `handleUpdateProgram`** — Show success/error toast.
8. **Add toast to `handleUpdateSession`** + `handleCreateSession` — Show success/error toast.

### Phase B: Advanced Filters in ContactSelector (US3) — `ContactSelector.tsx`

1. **Add `advancedFilters` prop** to ContactSelectorProps interface
2. **Add advanced filter state** — tag names (include/exclude), tag mode, formula type/text, device, date fields
3. **Add `/api/contacts` fetch path** — When `advancedFilters && sourceFilter="contact"`, call `/api/contacts` with full params. Map Contact → PersonResult.
4. **Add devices fetch** — Load available devices for device filter dropdown
5. **Render 2-column filter panel** — Left: device dropdown + date presets. Right: tag include/exclude cycle + AND/OR toggle + FormulaEditor toggle.
6. **Update active filter count** and "Limpiar filtros" to include new filters

### Phase C: Wire + Deploy

1. **Pass `advancedFilters`** to ContactSelector in `programs/[id]/page.tsx`
2. **Build**: `docker compose build frontend`
3. **Deploy**: `docker compose up -d`
4. **Verify**: `docker compose logs --tail=30 frontend`
